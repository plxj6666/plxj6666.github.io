题目描述

给你一个非负整数数组 nums 和一个整数 target 。

向数组中的每个整数前添加 '+' 或 '-' ，然后串联起所有整数，可以构造一个 表达式 ：

  • 例如，nums = [2, 1] ，可以在 2 之前添加 '+' ，在 1 之前添加 '-' ，然后串联起来得到表达式 "+2-1" 。

返回可以通过上述方法构造的、运算结果等于 target 的不同 表达式 的数目。

算法思路

这道题目我的解题过程令人发笑，具体可以往下看……首先我就开始想摆弄一些数学公式，经过思考我发现：假如我们记nums的总和为total，如果我们此时有一个数字nums[i]不选了，那么很明显，此时得到的和为total-nums[i]，那么如果我们再给他加一个负号，总和就变成了total-nums[i]-nums[i] = total-2*nums[i]。其实说到这里已经有一些眉目了，我们展开说：

  1. 我们记nums的总和为total，现在需要取出一些数字给他加上负号，假定我们取出来的这些数字的和为q
  2. 那么很显然，此时的和已经变成了total - 2 * q ，如果我们想凑出target，那么只需要让total-2*q=target即可
  3. 变换等式得到q=(total-target)/2

由上面的思路可以看出，我们只需要从nums中取出一些数字，让其和等于q即可，相比于原问题就简单得多。

思路借鉴

当然，我的“数学公式”并非一无是处，如果用在dp上，就是一组好解。我们借鉴官方的动态规划思路，大概是这样的：

  1. 定义二维数组dp，dp[i][j]表示在数组的前i个元素中能凑成j的方案数量，则我们最终所要求的就是dp[n][q]（q在上文提到过）。
  2. 如果没有元素，也就是nums为空，那么很显然有如下边界条件：

  1. 当数组中存在元素，1 <= i <= n，对于数组nums中的第i个元素，需要遍历0 <= j <= q，计算dp[i][j]的值，也就是相当于考虑nums[i]时需要更新状态。此时又分如下两种情况：

    • 如果j < num[i]，则肯定不能选num[i]，此时有dp[i][j] = dp[i-1][j]（也就是考虑nums[i]没什么作用）。

    • 如果j >= num[i]，则可以选，可以不选，如果不选，则依旧有dp[i-1][j]种方案；如果选的话，我们需要在前i-1个元素中凑出j-num[i]，也就是dp[i-1][j-num]种，总共就有 dp[i][j]=dp[i-1][j]+dp[i-1][j-num]。
